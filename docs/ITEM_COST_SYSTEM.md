# アイテムコスト・スタック制限システム (Item Cost & Stack Restriction System)

このシステムは、アイテムごとにインベントリ内の「占有サイズ（コスト）」を設定し、特定のアイテム（弾薬・紙幣）以外のスタックを制限することで、ゲームバランスとリアリズムを向上させるためのものです。

## 1. アイテムコスト (Item Cost)

各アイテムは `cost` プロパティを持ち、インベントリ内でどれだけのスロットを「追加で」占有するかを決定します。

- **コスト1**: 1スロットのみ占有（通常のアイテム）。
- **コスト2以上**: アイテム自体のスロットに加えて、`(コスト - 1)` 分のスロットをインベントリの右下から占有します。
- **占有スロット**: 「サイズ制限」という名前のバリアブロックが自動的に配置されます。これらは操作・移動・ドロップが不可能です。

### 占有の優先順位
占有スロットは、プレイヤーが現在アクセス可能なストレージスロット（ホットバー + リグによって解放されたメインインベントリ）の**右下から順に**埋められます。アイテムが既に置かれているスロットはスキップされます。

## 2. スタック制限 (Stack Restriction)

アイテムの「溜め込み」を防止するため、以下のルールでスタック制限が適用されます。

- **弾薬 (AMMO)**: スタック可能。
- **紙幣 (MONEY)**: スタック可能。
- **それ以外**: 全てのアイテム（銃、医療品、装備品、アタッチメント等）は**スタック不可 (最大1個)** に強制されます。

## 3. 設定方法 (Configuration)

`src/main/resources/items/` 内の各アイテム定義 YAML ファイルで設定可能です。

```yaml
gun_ak74:
  type: "GUN"
  material: "CROSSBOW"
  displayName: "AK-74"
  cost: 6          # この銃は合計6スロット分を占有する
  stackable: false # スタック不可（デフォルト）
  # ...
  
ammo_545x39:
  type: "AMMO"
  material: "IRON_NUGGET"
  cost: 1          # 弾薬は通常1スロット
  stackable: true  # 弾薬はスタック可能
```

※ `type` が `AMMO` または `MONEY` の場合、`stackable` は自動的に `true` に設定されます。

## 4. 技術的仕様 (Technical Specifications)

### 関連クラス
- **`CostSlotManager`**: インベントリの合計コストを計算し、占有スロットを動的に再配置するコアロジック。
- **`ItemFactory`**: アイテム生成時にコスト情報を PDC (`item_cost`) に埋め込み、スタックサイズを制限。
- **`InventoryListener`**: クリック、ドロップ、ピックアップ、インベントリを閉じる等のアクション時に `CostSlotManager` を呼び出して同期。
- **`RigModule` / `BackpackModule`**: リグによるメインインベントリの解放状態や、バックパック内インベントリのコスト計算をサポート。

### データの整合性
- 占有スロット（プレースホルダー）は、バックパックやデータベースなどの**保存データには含まれません**。
- インベントリをロードした際や、定期的（5ティックごと）なタスクによって、現在のアイテム構成に基づき占有スロットが常に再計算されます。これにより、アイテム消失やスロットが埋まったままになるバグを防止しています。
